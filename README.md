# lyrics

CLI-скрипт для автоматичного вбудовування текстів пісень у теги аудіофайлів (.mp3, .flac, .m4a).

## Що робить

- Читає artist/title з тегів аудіофайлу
- Шукає текст по ланцюжку джерел: **LRCLIB** → **Genius**
- Записує plain-текст у стандартні теги (`USLT` для MP3, `LYRICS` для FLAC, `©lyr` для M4A)
- Неточні збіги відкладає в чергу на ручне підтвердження з каскадом по джерелах

## Встановлення

```bash
git clone https://github.com/onilyxe/lyrics.git
cd lyrics
uv venv
uv pip install -r requirements.txt
```

## Genius API токен

1. Зареєструйся на [genius.com/api-clients](https://genius.com/api-clients)
2. Створи новий додаток
3. Скопіюй Client Access Token
4. Встав його у `lyrics.py` в рядок `GENIUS_TOKEN = "..."`

Без токена скрипт працює, але використовує лише LRCLIB (джерела 1 і 2 з 3).

## Використання

```bash
cd /шлях/до/музики

uv run python /шлях/до/lyrics.py

uv run python lyrics.py --dry-run

uv run python lyrics.py --overwrite
```

| Прапорець | Що робить |
|-----------|-----------|
| `--dry-run` | Лише показати що знайдеться, не записувати |
| `--overwrite` | Перезаписувати файли, де вже є текст |

## Джерела текстів

| # | Джерело | API-ключ |
|---|---------|----------|
| 1 | [LRCLIB](https://lrclib.net) exact | не потрібен |
| 2 | LRCLIB search | не потрібен |
| 3 | [Genius](https://genius.com) | потрібен токен |

## Як працює ревʼю

Якщо скрипт не впевнений що знайшов правильний текст — трек відкладається в чергу. Після обробки всіх файлів показується інтерактивне ревʼю:

- **y** — записати текст
- **n** — спробувати наступне джерело
- **s** — пропустити трек повністю
- **q** — вийти з ревʼю

## Куди пишуться теги

| Формат | Тег |
|--------|-----|
| MP3 | `USLT` (ID3v2) |
| FLAC | `LYRICS` (Vorbis Comment) |
| M4A | `©lyr` (MP4 atom) |
